# Code of Conduct

CivicSec Lab is a public-interest technology project. Contributors are expected to help make the project welcoming, careful, and useful for people with different levels of technical experience.

## Expected Behaviour

- Be respectful and constructive in discussions.
- Explain security, privacy, and data concepts without gatekeeping.
- Keep feedback focused on the work and the project goals.
- Respect the responsible-use boundaries of the project.
- Avoid publishing real sensitive data in issues, pull requests, examples, screenshots, or demo material.

## Unacceptable Behaviour

- Harassment, threats, personal attacks, or discriminatory language.
- Attempts to add exploit code, credential harvesting, malware, offensive automation, or targeting features.
- Publishing secrets, real personal data, or sensitive operational details.
- Pressuring maintainers or contributors to weaken safety boundaries.

## Reporting

Report conduct concerns through the project maintainers. If the concern involves a security issue, follow [SECURITY.md](SECURITY.md).

Maintainers may remove comments, close issues, reject contributions, or block participants when behaviour conflicts with this code of conduct.
