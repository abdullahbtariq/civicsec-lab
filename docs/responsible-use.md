# Responsible Use

CivicSec Lab is for defensive, educational, and public-interest work.

The platform should help civic organisations understand risk signals without encouraging exploitation, surveillance, harassment, or automated targeting.

## Product Rules

- Do not include exploit code.
- Do not include credential harvesting.
- Do not include malware, stealth, evasion, persistence, or privilege escalation logic.
- Do not create tools for targeting private individuals.
- Use fictional sample data by default.
- Make uncertainty and confidence visible.
- Require human review for risk interpretation.

## Module Notes

- ThreatBoard should prioritise defensive vulnerability triage.
- LogLens should use authorised or synthetic login logs.
- DataPrivacy Doctor should minimise exposure of raw personal data.
- Misinformation Observatory should focus on aggregate narrative signals.
- Civic Risk Graph should connect evidence without overclaiming certainty.
- IncidentFlow should support documentation and response, not retaliation.
